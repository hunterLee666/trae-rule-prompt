# 全链路主动触发机制 v4.0

> 解决：1) 触发关键词太少  2) 只依赖用户消息触发
> 目标：在整个对话生命周期的每个节点都能自动触发对应技能

---

## 🔄 触发生命周期矩阵

| 触发时机 | 检测内容 | 自动激活技能 |
| :--- | :--- | :--- |
| **用户发消息后** | 用户意图识别 | ✅ 第1层触发 |
| **AI 回复前** | 回复意图预测 | ✅ 第2层触发 |
| **工具调用前** | 工具调用分析 | ✅ 第3层触发 |
| **工具调用后** | 结果模式匹配 | ✅ 第4层触发 |
| **每 N 轮对话** | 上下文状态检测 | ✅ 第5层触发 |

## 🎯 第5层：周期性状态触发

**每 3-5 轮对话自动触发：**

| 检查项 | 阈值 | 自动操作 |
| :--- | :---: | :--- |
| **待办完成率** | < 30% | 调用 chief-of-staff 做任务梳理 |
| **上下文预算** | < 20% | 调用 context-budget 优化 |
| **代码行数** | > 200 行 | 调用 code-reviewer 评审 |
| **测试失败** | > 3 次 | 调用 harness-optimizer 分析 |
| **同工具重试** | ≥ 3 次 | 调用 loop-operator 破局 |

## ⚡ 实现机制

### 1. 规则注入方式
```
【每次回复前静默执行】

def auto_trigger(context):
    # 第1层：用户消息触发
    trigger_by_user_message(context)
    
    # 第2层：回复前预判触发
    trigger_by_intention_prediction(context)
    
    # 第3层：工具调用前拦截
    trigger_before_tool_call(context)
    
    # 第4层：工具结果触发
    trigger_after_tool_call(context)
    
    # 第5层：周期性触发
    if context.turn_count % 3 == 0:
        trigger_periodic_check(context)
```

### 2. 无感知激活
> 用户完全不需要知道这个机制的存在
> 所有技能调用和规则注入都在后台静默执行
> 只在真正需要用户决策时才出现

## 🎯 权责锁定全程生效

无论哪个触发层，都**强制遵守** CLAUDE.md 权责锁定：

| 链路 | 独占领域 | 越界拦截 |
| :--- | :--- | :--- |
| **Superpowers** | TDD 单元测试、核心代码评审 | ❌ 禁止生成 Playwright 代码 |
| **ECC** | E2E 测试、安全扫描、性能优化 | ❌ 禁止编写单元测试 |
