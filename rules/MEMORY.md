# MEMORY

> 持久记忆系统 - 跨会话记忆项目状态

---

## project-state.md

**位置**: 项目根目录 `.trae/project-state.md`

**作用**: 跨会话记忆项目状态

### 内容结构

```markdown
# 项目状态

## 项目概览
- 项目名称: XXX
- 技术栈: React + Node + PostgreSQL
- 架构: 微服务

## 待办事项
- [ ] 用户认证模块未完成
- [ ] API 文档待更新

## 技术决策
- 状态管理: Redux Toolkit
- CSS: Tailwind

## 历史问题
- #23: 缓存穿透问题已解决
- #45: 性能瓶颈已优化

## 最近变更
- 2026-03-09: 添加用户模块
```

### 记忆更新规则

| 操作 | 更新记忆？ |
|------|----------|
| 新功能开发 | ✅ 记录架构变更 |
| Bug 修复 | ✅ 记录问题和解决方案 |
| 技术选型 | ✅ 记录决策和理由 |
| 待办事项 | ✅ 实时更新 |
| 代码修改 | ❌ 不需要 |

---

## 启动流程

```
每次用户启动对话:
    ↓
1. 读取 .trae/project-state.md
    ↓
2. 了解项目当前状态
    ↓
3. 检查待办事项
    ↓
4. 主动提示用户:
   "项目当前状态: 
   - 待处理: 3 项
   - 最近变更: XXX
   需要继续之前的任务吗？"
```

---

## 心跳机制

**不是真正的时间心跳，而是"会话心跳"**:

| 触发条件 | 行为 |
|----------|------|
| 用户首次对话 | 读取 project-state，汇报状态 |
| 任务完成 | 主动检查相关工作 |
| 检测到问题 | 不仅修复，还检查同类问题 |
| 长时间无操作 | 提醒待办事项 |

### 状态汇报模板

```
用户: [无输入或开始新对话]

→ 读取 project-state
→ 汇报:
"""
📊 项目状态:
- 待办: 3 项
- 最近: 完成了用户模块
- 提醒: API 文档需要更新

需要我做什么？
"""
```

## 主动检查清单

```
任务完成后，自动检查:
├── [ ] 是否有相关测试需要更新？
├── [ ] 是否有文档需要同步？
├── [ ] 是否有类似问题需要修复？
├── [ ] 是否需要更新 project-state？
└── [ ] 是否有依赖需要更新？
```
