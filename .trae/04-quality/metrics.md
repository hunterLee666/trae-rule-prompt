# 质量指标

> **"可度量，方可改进。"**

## 一、代码质量指标

```yaml
复杂度指标:
  圈复杂度:
    目标: < 10
    警告: 10-20
    错误: > 20
    
  认知复杂度:
    目标: < 15
    警告: 15-25
    错误: > 25

可维护性指标:
  代码行数:
    函数: < 50行
    类: < 500行
    文件: < 1000行
    
  重复率:
    目标: < 3%
    警告: 3-5%
    错误: > 5%
```

## 二、测试质量指标

```yaml
覆盖率指标:
  行覆盖率:
    目标: > 80%
    警告: 60-80%
    错误: < 60%
    
  分支覆盖率:
    目标: > 70%
    警告: 50-70%
    错误: < 50%

测试效果指标:
  测试通过率:
    目标: 100%
    
  缺陷检出率:
    目标: > 90%
    
  回归测试通过率:
    目标: 100%
```

## 三、过程质量指标

```yaml
开发效率:
  代码审查通过率:
    目标: > 80%
    
  一次提交成功率:
    目标: > 70%
    
  平均修复时间:
    目标: < 1天

缺陷指标:
  缺陷密度:
    目标: < 1个/千行
    
  缺陷修复率:
    目标: > 95%
    
  缺陷重开率:
    目标: < 5%
```

## 四、指标收集

```yaml
收集方式:
  自动收集:
    - CI/CD流水线
    - 静态分析工具
    - 测试框架
    
  手动收集:
    - 代码审查
    - 用户反馈
    - 问题跟踪

收集频率:
  实时: CI/CD结果
  每日: 质量报告
  每周: 趋势分析
  每月: 综合评估
```

## 六、指标应用

```yaml
质量门禁:
  - 覆盖率不达标禁止合并
  - 复杂度超标需要审查
  - 缺陷未修复禁止发布

持续改进:
  - 定期分析指标趋势
  - 识别质量瓶颈
  - 制定改进计划
  - 追踪改进效果
```
